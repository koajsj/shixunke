汽车租赁系统答辩讲解稿 + 分层说明

一、项目简介

这个项目是一个汽车租赁系统的简化版管理平台，主要完成客户、车辆、租赁记录、支付记录这四类核心数据的管理。
项目采用 Spring Boot 开发后端，前端使用原生 HTML、CSS、JavaScript，实现了一个结构清晰、便于演示和讲解的小型管理系统。

系统当前可以完成的主要功能有：
1. 查看客户、车辆、租赁、支付数据
2. 新增客户、车辆、租赁、支付数据
3. 按业务规则校验数据，例如：
   - 车牌号不能重复
   - 客户手机号、邮箱、驾驶证号不能重复
   - 租赁日期不能冲突
   - 支付金额不能超过订单总金额


二、技术栈

本项目使用的技术栈比较基础，也比较适合课程答辩和讲解：

1. 后端框架：Spring Boot
2. Web 层：Spring MVC
3. 持久层：Spring Data JPA
4. 数据库：默认使用 H2 文件数据库，同时支持 MySQL
5. 数据初始化：car.sql
6. 前端页面：HTML + CSS + JavaScript
7. 测试：JUnit + Spring Boot Test


三、项目分层说明

这个项目采用了比较标准的分层结构，主要分为五层。

1. 基础层

基础层主要负责数据库和项目运行配置。

包括：
- car.sql：数据库建表和初始化数据脚本
- application.properties：数据源、JPA、SQL 初始化等配置

这一层的作用是：
- 定义数据库表结构
- 配置系统如何连接数据库
- 配置项目启动时自动执行 SQL 脚本


2. 持久层

持久层主要由 entity 和 repository 组成。

entity：
- 对应数据库中的表
- 例如 Customer、Car、Rental、Payment
- 作用是把数据库中的表映射为 Java 对象

repository：
- 负责访问数据库
- 继承 JpaRepository
- 不需要手写大量 SQL，就可以完成常见的增删改查

这一层的作用是：
- 完成对象和表之间的映射
- 提供数据访问能力


3. 业务层

业务层主要由 service 接口和 service 实现类组成。

service/interfaces：
- 定义业务方法
- 例如查询、创建、修改、删除

service/impl：
- 实现具体业务逻辑
- 比如唯一性校验、状态校验、金额校验、租赁冲突校验

这一层是项目的核心层。
因为真正的业务规则都放在这里，而不是直接写在控制器里。

这样做的好处是：
- 结构清晰
- 业务逻辑集中
- 便于维护和讲解


4. 控制层

控制层就是 controller 层。

例如：
- CarController
- CustomerController
- RentalController
- PaymentController

控制层主要负责：
- 接收前端请求
- 调用 service 处理业务
- 返回 JSON 数据给前端

控制层本身尽量保持简单，不写复杂业务逻辑。
这样符合“控制层薄、业务层厚”的设计思路。


5. 表示层

表示层就是前端页面，放在 static 目录下。

包括：
- index.html
- styles.css
- app.js

这一层负责：
- 页面展示
- 表单输入
- 调用后端接口
- 展示返回的数据和错误信息

这里没有使用 Vue，而是使用原生前端技术。
这样做的原因是：
- 项目更轻量
- 更容易理解
- 更适合课程展示和答辩讲解


四、系统请求执行流程

这个项目的请求流程可以概括为：

页面 -> Controller -> Service -> Repository -> 数据库

可以这样解释：

1. 用户在网页上填写表单并点击新增
2. 前端 JavaScript 发起 HTTP 请求
3. Controller 接收请求参数
4. Controller 调用对应的 Service
5. Service 进行业务校验和处理
6. Repository 负责把数据保存到数据库
7. 最后把结果返回给前端页面显示

这套流程体现了前后端分工和后端分层设计。


五、核心业务规则举例

为了说明这个项目不是简单的增删改查，我会重点讲几个业务规则。

1. 客户信息唯一性校验

客户的手机号、邮箱、驾驶证号都不能重复。
这样可以避免一人重复注册，保证数据有效性。


2. 车辆状态校验

车辆有可租、已租、维修等状态。
只有符合条件的车辆才能创建有效租赁记录。


3. 租赁时间冲突校验

同一辆车在同一时间段内不能被重复租赁。
系统会检查租赁日期是否和已有租赁记录冲突。


4. 支付金额校验

支付金额不能超过该租赁订单的总金额。
如果支付总额达到订单金额，系统会更新相应的租赁状态。


5. 完成态和取消态限制

已经完成或已取消的租赁记录，系统会限制部分不合理操作。
这样可以保证业务数据状态一致。


六、为什么这样设计

我在设计时重点考虑了三点：

1. 结构清晰

项目必须有明显的分层，这样代码职责明确，后续维护也方便。


2. 功能完整但不过度复杂

系统保留了汽车租赁的核心实体和核心业务规则，但没有引入过多复杂模块。
这样既能体现项目完整性，也方便讲解。


3. 适合本地演示

项目启动后可以直接在浏览器访问页面，不需要再单独部署前端。
老师可以直接看到页面、接口和数据库之间的联动效果。


七、项目亮点

我认为这个项目有以下几个亮点：

1. 分层完整，结构规范
2. 前后端联通，可以直接演示
3. 保留了真实业务规则，不只是基础 CRUD
4. 使用 JPA 简化持久层代码，提高开发效率
5. 前端保持轻量，便于理解和讲解


八、项目可改进点

如果后续继续完善，这个项目还可以增加：

1. 网页端编辑和删除功能
2. 下拉选择客户、车辆、租赁，而不是手动输入 ID
3. 登录权限管理
4. 更完整的统计报表
5. 使用 Vue 或其他前端框架进一步增强交互体验

十一、如果要用 MySQL 和 Navicat 查看数据

项目已经保留了 JPA，同时新增了 MySQL 配置文件：
- src/main/resources/application-mysql.properties

使用方式：
1. 先在本机安装并启动 MySQL
2. 确认账号密码，默认配置是：
   - 用户名：root
   - 密码：123456
3. 用下面命令启动项目：
   - mvnw.cmd spring-boot:run "-Dspring-boot.run.profiles=mysql"
4. 启动后项目会连接本地 MySQL，并自动执行 car.sql
5. 然后在 Navicat 中连接本地 MySQL，查看 car_rental 数据库即可

如果你的 MySQL 用户名或密码不同，也可以通过环境变量覆盖：
- DB_URL
- DB_USERNAME
- DB_PASSWORD
